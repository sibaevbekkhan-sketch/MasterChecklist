# AI OS: Terminal Rules (TERMINAL_RULES.md)
*Инструкции по генерации, подготовке и выполнению команд командной строки (DevOps Layer). Обеспечивают мгновенную работоспособность (Zero-Friction Copy-Paste).*

---

## 1. Главный Стандарт Командной Строки (Zero-Friction Command Execution)
Пользователь должен иметь возможность скопировать любую команду из ответа AI и выполнить её в своем терминале без необходимости ручного редактирования.

### Строжайше запрещено использовать плейсхолдеры и абстрактные маркеры:
* `<путь>`, `<path_to_file>`
* `<имя>`, `<filename>`, `<username>`
* `замени ... на ...`
* `подставь свои данные`
* `измени самостоятельно`
* `отредактируй строку перед запуском`

Каждая сгенерированная команда должна быть **полностью функциональна** и содержать **реальные, актуальные для текущего окружения пути и параметры**.

---

## 2. Протокол "Динамического Исследования" (Dynamic Discovery Protocol)
Если для формирования команды необходимы данные об окружении (пути, файлы, установленное ПО), AI обязан выполнить предварительную диагностику **до** того, как покажет команду пользователю.

### Алгоритм Dynamic Discovery:
1. **Неизвестен путь/файл**: Использовать инструменты поиска (например, `glob`, `list_directory` или `grep_search`), чтобы локализовать точное расположение целевого файла или директории в файловой системе.
2. **Неизвестна программа/зависимость**: Выполнить проверку наличия утилиты в системе (например, `Get-Command` в PowerShell или `which` в Unix) перед выводом команды, использующей эту утилиту.
3. **Неизвестен сетевой порт**: Автоматически проверить занятость портов или предложить команду, динамически определяющую свободный порт.

---

## 3. Совместимость со Средой Исполнения (Shell Compatibility)
Генерируемые команды должны строго соответствовать текущей операционной системе пользователя (для текущего контекста: **Windows (win32) / PowerShell**):
* Использовать корректный синтаксис переменных PowerShell (например, `$env:VAR_NAME` вместо `$VAR_NAME`).
* Использовать правильные разделители путей (`\` для Windows, `/` для Unix, либо генерировать кроссплатформенные пути).
* Использовать встроенные командлеты PowerShell (например, `Copy-Item`, `Test-Path`, `New-Item`) вместо Unix-команд `cp`, `test`, `touch` (если не установлены эмуляторы), чтобы избежать синтаксических ошибок парсинга.
* При вызове внешних утилит (например, `npm`, `python`, `git`) применять флаги подавления интерактивного ввода (например, `--yes`, `-y`, `--quiet`, `--no-pager`), чтобы команды завершались автоматически без ожидания ввода от пользователя.

---

## 4. Конкатенация и Оптимизация Команд
* Чтобы минимизировать количество ручных действий со стороны пользователя, объединяйте связанные команды в одну цепочку с помощью логических операторов (например, `&&` в CMD или `;` / `and` в PowerShell), если это безопасно и логично.
* **Пример правильной команды (PowerShell)**:
  ```powershell
  # Проверка существования директории и создание файла с последующей сборкой
  if (!(Test-Path -Path "C:\AI\AI_RULES")) { New-Item -ItemType Directory -Path "C:\AI\AI_RULES" }; cd "C:\AI\AI_RULES"; npm run build
  ```
  *(Все пути определены, создание директории безопасно, переходы и сборка объединены в один шаг)*.
